# Medicine Catalog Sync System - MedRush\n\n## Overview\nThis document explains how the medicine catalog system synchronizes between the **Admin Panel** and the **Customer Page**.\n\n---\n\n## How It Works\n\n### 1. **Admin Adding/Updating Medicines**\n\nWhen an admin adds or updates medicines:\n\n- **File**: `admin/medicines.html` & `admin/medicines.js`\n- **Storage Key**: `localStorage` with key `'medicineCatalog'`\n- **Data Structure**:\n  ```javascript\n  {\n    id: 1,\n    name: \"Medicine Name\",\n    category: \"pain-relief\", // or vitamins, cold-cough, antibiotics, diabetes\n    price: 100.00,\n    stock: 50,\n    isAvailable: true,\n    status: \"active\", // or inactive\n    description: \"Medicine description\",\n    type: \"capsule\",\n    meta: \"strip of 10 tablets\",\n    manufacturer: \"Various\",\n    composition: \"Medicine Name\",\n    createdAt: \"2026-04-14T12:00:00Z\"\n  }\n  ```\n\n### 2. **Customer Viewing Medicine Catalog**\n\nWhen a customer views the medicine page:\n\n- **File**: `customer/medicine.html` & `customer/medicine.js`\n- **Storage Key**: Same `'medicineCatalog'` key\n- **Features**:\n  - Loads medicines from localStorage\n  - Displays all **active** medicines with **stock > 0**\n  - Shows **category badges** (Pain Relief, Vitamins, etc.)\n  - Displays medicine details: name, price, stock, composition, manufacturer, description\n\n---\n\n## Real-Time Synchronization\n\n### Storage Events\nBoth pages listen for storage changes:\n\n```javascript\n// In customer/medicine.js\nwindow.addEventListener('storage', (event) => {\n    if (event.key === MEDICINE_STORAGE_KEY) {\n        medicines = loadMedicineCatalog();\n        updateMedicines();\n    }\n});\n```\n\n### Periodic Polling\nCustomer page checks for new medicines every 3 seconds:\n\n```javascript\nsetInterval(() => {\n    const updatedMedicines = loadMedicineCatalog();\n    if (JSON.stringify(updatedMedicines) !== JSON.stringify(medicines)) {\n        medicines = updatedMedicines;\n        updateMedicines();\n    }\n}, 3000);\n```\n\n---\n\n## Categories Supported\n\n1. **pain-relief** - 💊 Pain Relief\n2. **vitamins** - 💉 Vitamins\n3. **cold-cough** - 🤒 Cold & Cough\n4. **antibiotics** - ⚕️ Antibiotics\n5. **diabetes** - 🩹 Diabetes\n\n---\n\n## Display on Customer Page\n\nEach medicine card displays:\n\n- ✅ **Category Badge** (e.g., \"PAIN RELIEF\")\n- 🏷️ **Medicine Name**\n- 💰 **Price** (in ₹)\n- 📦 **Stock Quantity** (if available)\n- 🏭 **Manufacturer** (if provided)\n- 🧪 **Composition** (if provided)\n- 📝 **Description** (if provided)\n- 📋 **Type & Packaging** (e.g., \"strip of 10 tablets\")\n- 🛒 **Add to Cart Button** (if in stock)\n\n---\n\n## Step-by-Step: Adding Medicine in Admin\n\n1. Login to Admin Panel\n2. Go to **Medicines** section\n3. Fill in the form:\n   - **Medicine Name**: (e.g., \"Paracetamol 500mg\")\n   - **Category**: Select from dropdown (e.g., \"Pain Relief\")\n   - **Price**: Enter price in Rs (e.g., 45.00)\n   - **Stock Quantity**: Enter quantity (e.g., 150)\n   - **Description**: (Optional)\n4. Click **\"Add Medicine\"**\n5. You'll see: \"Medicine added successfully! It will appear on the customer page\"\n\n---\n\n## Step-by-Step: Viewing on Customer Page\n\n1. Open **Medicine** section on customer page\n2. Scroll through medicines or use search\n3. Each medicine displays with:\n   - Category badge (colored)\n   - Full details and price\n   - Stock status\n   - Add to Cart button\n\n---\n\n## Technical Details\n\n### Storage Mechanism\n- **Type**: Browser localStorage\n- **Key**: `'medicineCatalog'`\n- **Scope**: Same domain (both admin and customer at `localhost`)\n- **Persistence**: Data persists even after browser refresh\n\n### Cross-Tab Sync\n- When admin opens multiple tabs, all tabs stay in sync\n- When customer opens multiple tabs, all tabs stay in sync\n- Cross-site communication uses `localStorage` events\n\n### Fallback Data\nIf no medicines exist in storage:\n- Customer page loads with default sample medicines\n- Admin page loads with empty table\n\n---\n\n## Important Notes\n\n1. **Availability**\n   - Medicine is shown only if: `isAvailable === true` AND `stock > 0` AND `status === 'active'`\n\n2. **Stock Updates**\n   - When stock becomes 0, medicine is marked as \"OUT OF STOCK\"\n   - When stock > 0 again, medicine reappears\n\n3. **Category Filtering**\n   - Admin can filter medicines by category\n   - Customer page displays category badges for easy identification\n\n4. **Real-Time Updates**\n   - Admin changes appear almost instantly on customer page\n   - Polling every 3 seconds ensures no medicine is missed\n\n---\n\n## Troubleshooting\n\n### Medicine not appearing on customer page?\n1. Check if medicine has **stock > 0**\n2. Check if medicine **status** is \"active\"\n3. Check if **isAvailable** is true\n4. Clear browser localStorage and refresh\n\n### Stock not updating?\n1. Refresh the customer page\n2. Check admin panel - verify stock was saved\n3. Check browser console for errors\n\n### Category not displaying?\n1. Verify admin filled in the category dropdown\n2. Check if category is one of the supported categories\n3. Clear cache and refresh page\n\n---\n\n## Future Enhancements\n\n- [ ] Backend API integration for persistent storage\n- [ ] Real-time WebSocket updates\n- [ ] Medicine images/icons per category\n- [ ] Inventory alerts for low stock\n- [ ] Medicine expiry date tracking\n- [ ] Batch/batch number management\n- [ ] Supplier information\n- [ ] Cost price vs selling price tracking\n\n---\n\n## Files Modified\n\n1. `admin/medicines.js` - Enhanced to save complete medicine data\n2. `customer/medicine.js` - Updated to load and display admin medicines\n3. `customer/medicine.css` - Added category badge styling\n4. `admin/admin.js` - Integrated AuthBridge\n5. `customer/auth.js` - Added AuthBridge and ConnectionManager\n\n---\n\n## Testing the Feature\n\n### Test Case 1: Add Medicine\n1.
